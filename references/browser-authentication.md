# 授权浏览器登录

## 流程

1. 为每个 `account_id` 创建独立 Playwright Browser Context。
2. 打开创作服务平台登录页面并向用户展示二维码。
3. 由账号所有者在手机端扫码并确认。
4. 检测验证码、风险验证或账号异常；出现时立即暂停。
5. 读取可见账号名称，并与 `assets/accounts.yaml` 的期望身份比较。
6. 严格身份模式下不匹配则返回 `ACCOUNT_MISMATCH`，不保存会话。
7. 登录成功后导出 storage state，以 AES-256-GCM 加密保存。
8. 支持状态检查、会话过期、退出和彻底删除。

## 安全边界

- 不保存明文密码。
- 不读取短信验证码，不自动确认扫码，不绕过风险验证。
- Cookie、Local Storage 和会话密钥不得写入日志或版本库。
- 每个租户和账号必须隔离会话；生产密钥应由 KMS 或系统钥匙串管理。
