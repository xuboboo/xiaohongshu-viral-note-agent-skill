# 受控发布

## 发布模式

- `DRAFT_ONLY`：填写并保存草稿，不点击发布。
- `REQUIRE_CONFIRMATION`：默认模式，生成预览和内容 Hash，用户批准后发布。
- `SCHEDULED_AUTO_PUBLISH`：用户批准具体内容和具体时间后执行。
- `FULLY_AUTOMATED`：默认关闭，只能由部署方显式开启。

## 发布步骤

1. 检查登录状态和目标账号身份。
2. 运行最终事实、原创、商业、合规和 AI 标识审核；真正发布前 `_preflight` 走 async 服务端 reverify（含可选语义 embedding 原创门禁）。
3. 计算内容 Hash，并创建短期审批 Token。
4. 区分图文与视频入口，上传媒体，可选上传封面。
5. 填写标题、正文、话题和地点，保存预览截图。
6. 发布前再次校验 Hash、审批有效期、每日上限和最小间隔。
7. 点击发布后检查结果页面，提取 URL 与笔记 ID，并写入审计记录。

## 阻断条件

验证码、风险验证、账号不匹配、未知页面、素材缺失、审批过期、内容改变、严重合规问题、疑似洗稿、未处理商业关系或 AI 标识、重复内容和达到发布上限都会阻断发布。

Redis 模式下使用 `scripts/run_scheduler.py` 运行持久化定时调度；单机模式使用进程内调度。
